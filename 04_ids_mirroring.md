# IDS Mirroring Notes

Этот файл про базовые идеи подключения IDS в учебном стенде.

---

## 1. Зачем отдельный mirror-интерфейс

Если IDS только наблюдает трафик, ее часто подключают отдельным интерфейсом:

- один интерфейс для управления
- второй интерфейс для захвата трафика

Что это дает:

- управление не смешивается с трафиком мониторинга;
- IDS не участвует в маршрутизации;
- стенд проще объяснять.

---

## 2. Поднять capture-интерфейс без IP

```bash
ip addr flush dev ens4
ip link set ens4 up
ip link set ens4 promisc on
```

Что делает:

- очищает адреса на интерфейсе;
- поднимает его;
- включает promiscuous mode.

Что реализует:

- возможность принимать зеркалируемые кадры без участия интерфейса в L3-маршрутизации.

---

## 3. Проверить режим интерфейса

```bash
ip link show ens4
```

Что делает:

- показывает состояние интерфейса.

Что реализует:

- проверку, что интерфейс действительно поднят и готов к захвату.

---

## 4. Генерация тестового трафика

```bash
for p in 22 80 81 443 445 3389; do timeout 1 bash -c "echo >/dev/tcp/203.0.113.20/$p" 2>/dev/null; done
```

или

```bash
nmap -sS -Pn 203.0.113.20
```

Что делает:

- создает набор попыток сетевого подключения к тестовому хосту.

Что реализует:

- трафик, который IDS должна увидеть.

---

## 5. Если есть tcpdump

```bash
tcpdump -ni ens4
```

Что делает:

- показывает, приходит ли на capture-интерфейс трафик.

Что реализует:

- быструю проверку до запуска полноценного IDS.
