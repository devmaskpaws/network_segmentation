# Firewall NAT DMZ

Этот файл объясняет базовые идеи firewall, NAT и публикации сервиса в `DMZ`.

---

## 1. Что такое DMZ

`DMZ` — это отдельный сегмент для приложений, к которым нужен доступ извне или из других сетей.

Зачем нужна `DMZ`:

- если сервис взломают, атакующий не окажется сразу во внутренней сети;
- можно отдельно фильтровать входящий и исходящий трафик;
- проще показывать архитектуру защиты.

---

## 2. Маскарадинг в Linux через iptables

Если учебный firewall собран на Linux:

```bash
sysctl -w net.ipv4.ip_forward=1
iptables -t nat -A POSTROUTING -o ens2 -j MASQUERADE
```

Что делает:

- включает IP forwarding;
- включает NAT для исходящего трафика через внешний интерфейс `ens2`.

Что реализует:

- выход внутренних узлов наружу через один адрес firewall.

---

## 3. Разрешить уже установленные соединения

```bash
iptables -A FORWARD -m conntrack --ctstate ESTABLISHED,RELATED -j ACCEPT
```

Что делает:

- пропускает ответный трафик уже существующих соединений.

Что реализует:

- корректную работу возвратного трафика без избыточно широких правил.

---

## 4. Разрешить MGMT к LAN

```bash
iptables -A FORWARD -s 198.51.100.0/24 -d 192.0.2.0/24 -p tcp -m multiport --dports 22,443 -j ACCEPT
```

Что делает:

- разрешает административный доступ из `MGMT` в `LAN` к типовым портам управления.

Что реализует:

- доступ администратора к внутренним сервисам без полного открытия всего сегмента.

---

## 5. Разрешить MGMT к DMZ

```bash
iptables -A FORWARD -s 198.51.100.0/24 -d 203.0.113.0/24 -p tcp -m multiport --dports 22,80,443 -j ACCEPT
iptables -A FORWARD -s 198.51.100.0/24 -d 203.0.113.0/24 -p icmp -j ACCEPT
```

Что делает:

- разрешает управление и проверку сервисов в `DMZ`.

Что реализует:

- контролируемый админский доступ к вынесенному приложению.

---

## 6. Разрешить LAN к DMZ

```bash
iptables -A FORWARD -s 192.0.2.0/24 -d 203.0.113.0/24 -p tcp -m multiport --dports 80,443 -j ACCEPT
iptables -A FORWARD -s 192.0.2.0/24 -d 203.0.113.0/24 -p icmp -j ACCEPT
```

Что делает:

- разрешает внутренним хостам обращаться к опубликованному приложению.

Что реализует:

- доступ внутренней сети к приложению в `DMZ` только по нужным протоколам.

---

## 7. Запретить WAN к LAN и MGMT

```bash
iptables -A FORWARD -i ens2 -d 192.0.2.0/24 -j DROP
iptables -A FORWARD -i ens2 -d 198.51.100.0/24 -j DROP
```

Что делает:

- блокирует прямой доступ извне к `LAN` и `MGMT`.

Что реализует:

- базовую изоляцию внутренних сегментов.

---

## 8. Опубликовать сервис из DMZ

```bash
iptables -t nat -A PREROUTING -i ens2 -p tcp --dport 80 -j DNAT --to-destination 203.0.113.20:80
iptables -A FORWARD -p tcp -d 203.0.113.20 --dport 80 -j ACCEPT
```

Что делает:

- перенаправляет входящий трафик на внешний `80/tcp` в `DMZ` на хост `203.0.113.20`;
- разрешает forwarding до этого хоста.

Что реализует:

- публикацию веб-сервиса через firewall.

---

## 9. Политика по умолчанию

```bash
iptables -P FORWARD DROP
```

Что делает:

- задает политику `DROP` для транзитного трафика.

Что реализует:

- подход `deny by default`, когда явно разрешается только нужное.

---

## 10. Что проверить после правил

```bash
iptables -S
iptables -t nat -S
```

Что делает:

- показывает текущие правила фильтрации и NAT.

Что реализует:

- аудит собственной настройки перед сдачей или тестом.
