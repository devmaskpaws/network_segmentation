# Validation Checklist

Короткий чек-лист для учебного стенда по сетевой сегментации.

---

## 1. Интерфейсы

Проверить:

- все нужные интерфейсы существуют
- все нужные интерфейсы подняты
- capture-интерфейс IDS без IP

Команды:

```bash
ip link show
ip addr show
```

---

## 2. IP-адресация

Проверить:

- `MGMT` хост находится в `198.51.100.0/24`
- `LAN` сервис находится в `192.0.2.0/24`
- `DMZ` сервис находится в `203.0.113.0/24`
- gateway настроены корректно

Команды:

```bash
ip addr show
ip route show
```

---

## 3. Связность

Проверить:

- admin host видит gateway
- admin host видит внутренний сервис
- admin host видит сервис в DMZ

Команды:

```bash
ping -c 4 198.51.100.1
ping -c 4 192.0.2.10
ping -c 4 203.0.113.20
```

---

## 4. Приложение

Проверить:

- HTTP или HTTPS реально отвечает

Команды:

```bash
curl http://203.0.113.20
curl -kI https://192.0.2.10
```

---

## 5. Firewall

Проверить:

- включен forwarding
- NAT есть
- есть правила между сегментами
- политика по умолчанию не слишком широкая

Команды:

```bash
sysctl net.ipv4.ip_forward
iptables -S
iptables -t nat -S
```

---

## 6. IDS

Проверить:

- интерфейс захвата поднят
- в IDS выбран правильный интерфейс
- тестовый трафик виден

Команды:

```bash
ip link show ens4
tcpdump -ni ens4
```

---

## 7. Общий критерий готовности

Стенд можно считать базово собранным, если:

- сети разделены на отдельные сегменты
- управление вынесено в отдельную сеть
- приложение вынесено в `DMZ`
- доступы ограничены правилами firewall
- есть подтверждаемая связность
- IDS получает тестовый трафик
