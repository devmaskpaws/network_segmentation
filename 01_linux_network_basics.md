# Linux Network Basics

Этот файл покрывает базовые команды, которые нужны почти в любом стенде.

---

## 1. Посмотреть интерфейсы

```bash
ip link show
```

Что делает:

- показывает сетевые интерфейсы и их состояние.

Зачем нужно:

- чтобы понять, как в системе называются сетевые карты, например `ens3`, `ens4`, `eth0`.

---

## 2. Посмотреть IP-адреса

```bash
ip addr show
```

Что делает:

- показывает IP-адреса на всех интерфейсах.

Зачем нужно:

- чтобы проверить текущую адресацию и убедиться, что нужный IP действительно назначен.

---

## 3. Поднять интерфейс

```bash
ip link set ens3 up
```

Что делает:

- переводит интерфейс `ens3` в состояние `UP`.

Зачем нужно:

- без этого интерфейс может быть выключен и сеть не заработает.

---

## 4. Удалить старый IP

```bash
ip addr flush dev ens3
```

Что делает:

- удаляет с интерфейса старые IP-адреса.

Зачем нужно:

- чтобы не оставить лишнюю адресацию и не получить конфликт маршрутизации.

---

## 5. Назначить IP-адрес

```bash
ip addr add 198.51.100.10/24 dev ens3
```

Что делает:

- назначает интерфейсу адрес `198.51.100.10/24`.

Зачем нужно:

- чтобы включить хост в нужный сетевой сегмент.

---

## 6. Добавить маршрут по умолчанию

```bash
ip route add default via 198.51.100.1
```

Что делает:

- указывает шлюз по умолчанию.

Зачем нужно:

- без default route хост будет видеть только свою локальную подсеть.

---

## 7. Посмотреть маршруты

```bash
ip route show
```

Что делает:

- показывает таблицу маршрутизации.

Зачем нужно:

- чтобы проверить, куда именно система будет отправлять трафик.

---

## 8. Проверить связность

```bash
ping -c 4 198.51.100.1
```

Что делает:

- отправляет ICMP-запросы узлу.

Зачем нужно:

- это самый быстрый способ проверить, жив ли шлюз или другой хост.

---

## 9. Проверить порт сервиса

```bash
curl http://203.0.113.20
```

или

```bash
curl -kI https://192.0.2.10
```

Что делает:

- проверяет, отвечает ли HTTP или HTTPS-сервис.

Зачем нужно:

- ping не показывает, работает ли именно приложение.
