# Segmentation And Routing

Этот файл показывает базовую учебную схему:

- `LAN` — внутренняя сеть сервисов
- `MGMT` — отдельная сеть управления
- `DMZ` — отдельная сеть для публикуемого приложения

---

## 1. Идея сегментации

Типовая схема:

- `LAN`: `192.0.2.0/24`
- `MGMT`: `198.51.100.0/24`
- `DMZ`: `203.0.113.0/24`

Почему это полезно:

- управление не смешивается с рабочим трафиком;
- приложение не размещается прямо во внутренней сети;
- можно ограничивать доступ между зонами правилами firewall.

---

## 2. Пример адресации

- gateway `LAN`: `192.0.2.1`
- gateway `MGMT`: `198.51.100.1`
- gateway `DMZ`: `203.0.113.1`
- admin host: `198.51.100.10`
- internal service: `192.0.2.10`
- dmz app: `203.0.113.20`

---

## 3. Настройка узла в MGMT

```bash
ip addr flush dev ens3
ip addr add 198.51.100.10/24 dev ens3
ip link set ens3 up
ip route add default via 198.51.100.1
```

Что реализует:

- подключение рабочей станции или admin host к сегменту управления.

---

## 4. Настройка узла в LAN

```bash
ip addr flush dev ens3
ip addr add 192.0.2.10/24 dev ens3
ip link set ens3 up
ip route add default via 192.0.2.1
```

Что реализует:

- размещение внутреннего сервиса в основной локальной сети.

---

## 5. Настройка узла в DMZ

```bash
ip addr flush dev ens3
ip addr add 203.0.113.20/24 dev ens3
ip link set ens3 up
ip route add default via 203.0.113.1
```

Что реализует:

- размещение публикуемого приложения в выделенной изолированной зоне.

---

## 6. Что должен делать маршрутизатор или firewall

На устройстве, которое стоит между сегментами, должны быть:

- интерфейс в `LAN`
- интерфейс в `MGMT`
- интерфейс в `DMZ`
- интерфейс во внешнюю сеть при необходимости

Что оно реализует:

- маршрутизацию между зонами;
- контроль доступа между зонами;
- NAT и публикацию сервисов.
